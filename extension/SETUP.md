# セットアップガイド

## Google Cloud Consoleでの設定手順

### 1. プロジェクトの作成

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセス
2. 画面上部の「プロジェクトを選択」をクリック
3. 「新しいプロジェクト」をクリック
4. プロジェクト名を入力（例：meeting-slot-generator）
5. 「作成」をクリック

### 2. Google Calendar APIの有効化

1. 左側メニューから「APIとサービス」→「ライブラリ」を選択
2. 検索ボックスに「Google Calendar API」と入力
3. 「Google Calendar API」を選択
4. 「有効にする」をクリック

### 3. OAuth同意画面の設定

1. 左側メニューから「APIとサービス」→「OAuth同意画面」を選択
2. ユーザータイプで「外部」を選択して「作成」をクリック
3. 必須項目を入力：
   - アプリ名：打ち合わせ候補日時生成
   - ユーザーサポートメール：あなたのメールアドレス
   - デベロッパーの連絡先情報：あなたのメールアドレス
4. 「保存して次へ」をクリック
5. 「スコープ」画面では何も追加せずに「保存して次へ」
6. 「テストユーザー」画面で自分のGoogleアカウントを追加
7. 「保存して次へ」をクリック

### 4. OAuth 2.0クライアントIDの作成

1. 左側メニューから「APIとサービス」→「認証情報」を選択
2. 「認証情報を作成」→「OAuthクライアントID」をクリック
3. アプリケーションの種類で「Chromeアプリ」を選択
4. 名前を入力（例：Meeting Slot Generator Extension）
5. アプリケーションIDを入力：
   - Chromeウェブストアで公開しない場合、拡張機能のIDを後で入力できます
   - 開発中は一時的な値として extension_id_placeholder を使用
6. 「作成」をクリック
7. 表示されたクライアントIDをコピー

### 5. manifest.jsonの更新

`extension/manifest.json`を開き、以下の部分を更新：

```json
"oauth2": {
  "client_id": "YOUR_CLIENT_ID.apps.googleusercontent.com",
  ...
}
```

`YOUR_CLIENT_ID`の部分を、コピーしたクライアントIDに置き換えます。

### 6. 拡張機能IDの取得と更新

1. Chromeで`chrome://extensions/`を開く
2. デベロッパーモードを有効化
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. `extension`フォルダを選択
5. 読み込まれた拡張機能のID（英数字の文字列）をコピー
6. Google Cloud Consoleに戻り、作成したOAuthクライアントIDを編集
7. アプリケーションIDに拡張機能のIDを入力
8. 「保存」をクリック

## トラブルシューティング

### 拡張機能IDが変わる場合

開発中は拡張機能のIDが変わる可能性があります。その場合：
1. 新しいIDを取得
2. Google Cloud ConsoleでOAuthクライアントIDのアプリケーションIDを更新

### 本番環境での使用

本番環境で使用する場合：
1. 拡張機能をChromeウェブストアに公開
2. 公開された拡張機能のIDを使用
3. OAuth同意画面を「公開」ステータスに変更

## セキュリティに関する注意

- クライアントIDは公開情報ですが、クライアントシークレット（Chrome拡張では使用しない）は秘密にしてください
- OAuth同意画面のスコープは必要最小限に保ってください
- テストユーザーを適切に管理してください
