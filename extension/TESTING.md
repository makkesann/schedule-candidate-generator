# テストガイド

## 手動テスト手順

### 1. 基本機能のテスト

#### 1.1 拡張機能の読み込み

- [ ] `chrome://extensions/`でデベロッパーモードを有効化
- [ ] 「パッケージ化されていない拡張機能を読み込む」でextensionフォルダを選択
- [ ] エラーが表示されないことを確認
- [ ] ツールバーにアイコンが表示されることを確認

#### 1.2 ポップアップUIの表示

- [ ] 拡張機能アイコンをクリック
- [ ] ポップアップが正しく表示されることを確認
- [ ] 以下の要素が表示されることを確認：
  - [ ] タイトル「打ち合わせ候補日時生成」
  - [ ] 候補数の入力フィールド（デフォルト: 5）
  - [ ] 会議時間の選択（デフォルト: 60分）
  - [ ] 抽出開始時間（デフォルト: 09:00）
  - [ ] 抽出終了時間（デフォルト: 18:00）
  - [ ] 曜日チェックボックス（月〜金がチェック済み）
  - [ ] 検索期間（デフォルト: 7日）
  - [ ] 「候補を生成」ボタン

### 2. Google認証のテスト

#### 2.1 初回認証

- [ ] 「候補を生成」ボタンをクリック
- [ ] Google認証画面が表示されることを確認
- [ ] アカウントを選択
- [ ] 権限の確認画面で「許可」をクリック
- [ ] 認証が成功することを確認

#### 2.2 認証エラーのハンドリング

- [ ] manifest.jsonのclient_IDを無効な値に変更
- [ ] 拡張機能を再読み込み
- [ ] 「候補を生成」をクリック
- [ ] エラーメッセージが表示されることを確認
- [ ] client_IDを正しい値に戻す

### 3. カレンダー取得のテスト

#### 3.1 予定がある場合

- [ ] Googleカレンダーに予定を追加（例：明日の10:00-11:00）
- [ ] 「候補を生成」をクリック
- [ ] 処理中メッセージが表示されることを確認
- [ ] 予定を避けた候補が生成されることを確認

#### 3.2 予定がない場合

- [ ] カレンダーに予定がない期間で実行
- [ ] 時間帯の全てが候補として提案されることを確認

#### 3.3 予定が多い場合

- [ ] カレンダーに多数の予定を追加
- [ ] 「候補を生成」をクリック
- [ ] 空き時間のみが候補として抽出されることを確認

### 4. 設定のテスト

#### 4.1 候補数の変更

- [ ] 候補数を1に設定
- [ ] 「候補を生成」をクリック
- [ ] 1件のみ出力されることを確認
- [ ] 候補数を10に設定
- [ ] 最大10件出力されることを確認

#### 4.2 会議時間の変更

- [ ] 会議時間を30分に設定
- [ ] 「候補を生成」をクリック
- [ ] 30分間の候補が生成されることを確認（例：10時00分〜10時30分）
- [ ] 会議時間を90分に設定
- [ ] 90分間の候補が生成されることを確認（例：10時00分〜11時30分）

#### 4.3 時間帯の変更

- [ ] 抽出開始時間を13:00に設定
- [ ] 抽出終了時間を18:00に設定
- [ ] 「候補を生成」をクリック
- [ ] 13:00〜18:00の範囲のみの候補が生成されることを確認

#### 4.4 曜日の変更

- [ ] 土日のみをチェック
- [ ] 「候補を生成」をクリック
- [ ] 土日の候補のみが表示されることを確認
- [ ] 全ての曜日のチェックを外す
- [ ] エラーメッセージが表示されることを確認

#### 4.5 検索期間の変更

- [ ] 検索期間を1日に設定
- [ ] 「候補を生成」をクリック
- [ ] 明日の候補のみが表示されることを確認
- [ ] 検索期間を30日に設定
- [ ] より多くの日付の候補が表示されることを確認

### 5. 設定の保存機能のテスト

#### 5.1 設定の永続化

### 6. 出力フォーマットのテスト

#### 6.1 日本語フォーマットの確認

生成された候補が以下の形式であることを確認：
- [ ] `MM月DD日（曜日）HH時MM分〜HH時MM分`
- [ ] 例：`3月15日（金）10時00分〜11時00分`
- [ ] 月は1桁または2桁（1月、12月）
- [ ] 日は1桁または2桁（1日、31日）
- [ ] 曜日は漢字1文字（日月火水木金土）
- [ ] 時間は1桁または2桁（9時、15時）
- [ ] 分は2桁（00分、30分）

#### 6.2 改行区切りの確認

- [ ] 複数の候補が改行で区切られていることを確認
- [ ] コピーした内容を貼り付けて、改行が正しく保持されることを確認

## デバッグ方法

### Chrome DevToolsの使用

1. **ポップアップのデバッグ**
   - ポップアップを右クリック → 「検証」
   - Console タブでエラーを確認
   - Networkタブで API リクエストを確認

2. **バックグラウンドスクリプトのデバッグ**
   - `chrome://extensions/` を開く
   - 拡張機能の「サービスワーカー」をクリック
   - Console でログを確認

3. **ストレージの確認**
   - DevTools → Application → Storage → Local Storage
   - chrome-extension://... の下に保存された設定を確認

### よくある問題と解決方法

| 問題 | 原因 | 解決方法 |
|------|------|----------|
| 認証エラー | client_IDが未設定 | manifest.jsonを確認 |
| 候補が生成されない | カレンダーが空 | Googleカレンダーに予定を追加 |
| クリップボードにコピーされない | 権限エラー | manifest.jsonにclipboardWriteが含まれているか確認 |
| 設定が保存されない | storage権限エラー | manifest.jsonにstorageが含まれているか確認 |
| UIが表示されない | CSSが読み込めない | ファイルパスを確認 |
